### 强制访问控制(MAC)的含义？

#### 2. BIBA完整性模型

- 下读上写，不可读低级数据，不可写高级数据，保证完整性

## 三、数字签名与身份认证

### 报文鉴别

- ***完整性包含报文源鉴别、报文宿鉴别、时间性鉴别***
- ***报文内容鉴别：报文鉴别码、报文加密、报文摘要***

- 报文鉴别码：用某个密钥生成小数据块追加在报文尾部
- 报文加密：完整的报文的密文作为鉴别码
- 报文摘要：将报文输入单向hash函数得到固定长度的输出作为报文摘要

### 散列函数

散列函数的基本性质：

1. 广泛使用性：适用于任何大小的数据分组
2. 码长固定性：不同输入产生定长输出
3. 易计算性：计算过程消耗资源少
4. 单向不可逆性：无法倒推
5. 弱单向性：给定X无法算出Y使得H(X)=H(Y)
6. 强单向性：找不出(X, Y)对使得H(X)=H(Y)

#### 1. MD5 

- 128位输出 
- 512位分组 划分为16个32位子分组
- 将输入的填充到512N+448位，最后尾部加上64位表示原来长度

## 四、网络安全协议

### 因特网与TCP/IP安全

#### 基于IP协议 

##### 1. Ping of death攻击

分片组装导致缓冲区溢出

##### 2. Teardrop攻击

修改片偏移使其小于上一篇结束的位移，造成数据包的重叠，合并数据过程导致堆栈溢出

### SSL协议

- 组成： 公开密钥体制和X.509数字证书技术
- 工作在传输层
- 四个协议：SSL握手协议、SSL告警协议、SSL修改密文协议、SSL记录协议
- 两个状态：连接状态、会话状态

##### 1. 会话状态要素

- 会话标识符、对方证书、压缩方法、密码规范、主密码、可重用否

##### 2. 连接状态要素

- 服务器和客户端的随机数、服务器端MAC密钥、客户端MAC密钥、服务器写密钥、客户端写密钥、初始化向量、序列号

##### 3. 发送过程

- 分片
- 压缩
- 加MAC
- 加密
- 加SSL记录报头

##### 4. 协商过程

- Hello Request from Server
- Client Hello(Client_version, Random, Session_id, Cipher_suite, Compression_methods)
- Server Hello(Server_version, Random, Session_id, Cipher_suite, Compression_methods) with Certificate (If there's an request from client)
- Server Hello Done

### IPSec协议

- 三要素：AH(96位+ICV)、ESP、IKMP
- AH提供数据源认证和无连接的数据完整性服务
- ESP在AH之上还可以提供无连接的数据保密服务

- AH、ESP是插入IP数据包的一个协议头，提供**数据完整性、数据源认证和抗重放攻击**等（*ESP*多了个**机密性**）
